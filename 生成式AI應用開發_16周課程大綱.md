# 生成式AI應用開發 18周課程大綱

## 一、課程基本資訊

- 課程名稱：生成式AI應用開發
- 授課週數：18週，含16週授課與2週自主學習
- 每週時數：3小時
- 授課對象：大學三年級資管系學生
- 課程取向：程式開發導向、實務應用導向
- 主要程式語言：Python
- 主要LLM平台：OpenAI API
- 補充比較平台：Claude API、Gemini API
- 主要Web App工具：Streamlit
- 不包含項目：本地端大型模型訓練、模型微調、高算力GPU實作

## 二、課程定位

本課程以 Python 與 LLM API 為核心，訓練學生開發可實際部署的生成式AI應用。課程不進行本地端模型訓練或微調，而是聚焦於 API 整合、Prompt 設計、Streaming 輸出、結構化輸出、文件處理、RAG、多模態圖片理解、Tool Calling、Agent 工作流程、Skill 融入應用開發、MCP 概念與 Web App 部署。

課程主線採用 OpenAI API，讓學生能在單一主要平台上完成完整開發流程；課程後段補充 Claude 與 Gemini，讓學生理解不同 LLM 平台的 API 差異、成本差異與應用場景差異。

## 三、課程目標

學生修完本課程後，應能：

1. 使用 Python 呼叫 OpenAI API 建立文字生成與對話應用。
2. 設計適合實務情境的 Prompt，並能控制輸出格式與品質。
3. 使用 Streaming 輸出改善聊天機器人與 Web App 的互動體驗。
4. 使用 Structured Outputs 將模型輸出轉為 JSON 或表格資料。
5. 處理 PDF、Word、CSV、圖片等資料來源。
6. 使用 Embeddings 與向量資料庫建立 RAG 文件問答系統。
7. 使用 Vision API 建立圖片理解、截圖分析與資料抽取應用。
8. 使用 Function Calling / Tool Calling 讓模型呼叫 Python 函式或外部工具。
9. 建立簡易 Agent 工作流程，完成多步驟任務。
10. 將上學期學過的 Skill 架構融入生成式AI應用開發，設計可重複使用的任務模組。
11. 理解 MCP 在 AI Agent 工具連接中的角色與應用情境。
12. 使用 Streamlit 建立並部署生成式AI Web App。
13. 理解 API 成本、資料隱私、幻覺、錯誤處理與安全風險。

## 四、授課方式

本課程採取「觀念講解、範例示範、課堂實作、專題整合」並重的方式進行，重點在於讓學生能逐步完成可部署的生成式AI應用。

| 授課方式 | 說明 |
|---|---|
| 觀念講解 | 說明 LLM API、Prompt、Structured Outputs、RAG、Vision、Tool Calling、Agent、MCP 等核心概念 |
| 程式示範 | 由教師示範 Python 呼叫 OpenAI API、處理資料、建立 Streamlit 介面與部署流程 |
| 課堂實作 | 每週安排實作練習，讓學生在課堂中完成可執行的小功能 |
| 分階段開發 | 前期以 Colab 降低環境設定負擔，後期轉為 VS Code 與本機專案開發 |
| 問題導向學習 | 以摘要、分類、文件問答、圖片理解、客服回覆、資料分析等實務問題作為練習情境 |
| 專題導向學習 | 期中完成個人小型 Web App，期末以小組方式完成可部署的 RAG / Agent / 多模態應用 |
| 同儕展示與回饋 | 透過期中展示、課堂 demo、期末發表與同儕回饋改善作品 |
| 自主學習 | 第17與第18週安排自主學習與作品修正，學生可依專題需求延伸 Claude / Gemini、MCP、FastAPI、資料庫或進階 RAG |

教學環境安排：

1. 第1週以瀏覽器與平台介面展示為主。
2. 第2至第6週以 Google Colab 或 Notebook 為主要範例環境。
3. 第7週開始導入 VS Code、本機 Python 環境、Streamlit 與 GitHub。
4. 第8週之後以 VS Code 與 GitHub 專案形式進行期中與期末專題開發。
5. 期末專題原則上需提供 GitHub repository、README 與可部署網址。

## 十一、Streaming 輸出教學定位

Streaming 輸出是生成式AI應用的重要互動設計，不應只放在補充內容。學生需理解：

- Streaming 可讓使用者逐步看到模型回覆，改善長回答等待體驗。
- Streaming 對聊天機器人、RAG 問答、Agent 任務回報與 Web App UX 很重要。
- Streaming 程式需要處理 token 片段累積、畫面即時更新、錯誤中斷與使用者取消等問題。
- Streaming 與 Structured Outputs 的使用情境不同：前者強調即時互動，後者強調格式穩定與系統整合。

本課程安排第5週先在 Python / Notebook 中理解串流輸出流程，第7週再整合到 Streamlit Web App，讓學生能在期中與期末專題中實際使用。

## 十二、MCP 教學定位

MCP 不作為本課程主線核心，但應納入第15週概念說明與示範。學生需理解：

- Tool Calling 是在自己的 App 中讓模型呼叫自訂函式。
- Agent 是模型根據任務目標，進行多步驟規劃、工具使用與結果整合。
- MCP 是一種讓 AI App 或 Agent 以標準方式連接外部工具與資料來源的協定。

本課程不要求所有學生從零撰寫 MCP Server，但期末專題若能使用 MCP 或設計類似 MCP 的工具連接架構，並清楚說明安全與權限控管，可列為進階加分項目。

| 40分鐘 | 專題部署測試、錯誤排除、成本與安全檢查 |

## 十三、Skill 融入應用開發定位

由於學生已在上學期「生成式AI理論與實務」中學過 Skill 架構，以及如何在 Claude Desktop 使用 Skill，本課程不再重複介紹 Skill 的基本概念與 Claude Desktop 操作流程。

本課程中的 Skill 重點改為「如何融入應用開發」：

- 將 Skill 視為可重複使用的任務模組，而不是單次 prompt。
- 將 Skill 的任務說明、輸入格式、輸出格式、工具使用規則與錯誤處理，轉化為 Python 程式中的模組設計。
- 在 Streamlit App 中讓使用者選擇不同任務模組，例如摘要 Skill、分類 Skill、資料抽取 Skill、客服回覆 Skill。
- 在 Agent 流程中讓模型依任務選擇適合的 Skill-like 模組與工具。
- 在期末專題中說明哪些功能被設計成可重複使用模組，以及這些模組如何被 Web App 或 Agent 呼叫。

建議第13週以一個簡單案例實作，例如「會議紀錄處理 Skill-like 模組」。教學上不將 Function Calling 與 Skill-like 模組設計分成兩段獨立內容，而是直接以 Skill-like 模組作為 Function Calling 的工具範例，讓學生在同一個實作中理解工具 schema、任務模組化與模型呼叫函式的關係。第14週則讓 Agent 呼叫前一週設計好的工具或模組，專心完成多步驟任務流程：

1. 輸入：會議逐字稿或會議筆記。
2. 任務：摘要重點、抽取決議、列出待辦事項。
3. 輸出：結構化 JSON 與使用者可讀摘要。
4. 工具：文字生成、Structured Outputs、必要時結合 Tool Calling。
5. 整合：在 Streamlit 介面中作為一個可重複使用功能模組。

## 十六、期末專題最低要求

每組期末專題應包含：

1. 一個可操作的 Streamlit Web App。
2. 至少一個 OpenAI API 功能。
3. 至少一種外部資料來源，例如 PDF、CSV、資料庫、圖片或公開 API。
4. 至少包含 RAG 或 Function Calling 其中一項。
5. 若做 Agent 題目，需呈現明確的工具選擇與多步驟流程。
6. GitHub repository。
7. 可部署網址。
8. README 說明功能、架構、使用方式、限制與成本估算。

進階加分項目：

- 多模態圖片理解，例如收據抽取、商品圖片分析、截圖診斷。
- Multi-agent 協作。
- 多模型比較。
- MCP 或類 MCP 工具連接架構。
- 使用者登入或權限控管。
- 資料庫儲存。
- Prompt 版本管理。
- API 成本紀錄。

## 十八、API與平台費用說明
