# 生成式AI應用開發 第01週教材

## 一、本週主題

課程導論與生成式AI應用架構

本週不要求學生撰寫程式，也不使用付費 API。重點是讓學生理解本課程要培養的能力、生成式AI應用的基本組成，以及後續 18 週會如何逐步完成一個可部署的 AI Web App。

## 二、學習目標

學生完成本週課程後，應能：

1. 說明本課程與「生成式AI理論與實務」的差異。
2. 區分 ChatGPT 類網頁工具與 LLM API 應用開發的差異。
3. 說明一個生成式AI應用的基本架構。
4. 初步理解 Prompt、Structured Outputs、RAG、Vision、Tool Calling、Agent、Skill、MCP、部署等概念在課程中的位置。
5. 理解本課程的工具、評量方式、期中與期末專題要求。
6. 理解 Git、GitHub 與 GitHub Desktop 在本課程專題開發中的用途。
7. 完成開發環境與帳號準備清單的確認。

## 三、課前準備

教師準備：

- 課程大綱 Markdown 或 PDF。
- OpenAI、Claude、Gemini 網頁版展示帳號。
- 一至兩個 AI Web App 範例，可使用公開 demo 或教師自製簡易展示。
- 電腦教室環境檢查清單。
- 學生背景調查表單，例如 Python 程度、是否有 GitHub 帳號、是否用過 ChatGPT API。

學生準備：

- 可登入學校電腦或自備筆電。
- 可使用瀏覽器。
- 建議先註冊 GitHub 帳號。
- 本週不需準備 API key。

## 五、投影片大綱

### Slide 1：課程名稱

生成式AI應用開發

副標題：

使用 Python、LLM API、RAG、Agent 與 Streamlit 建立可部署的 AI Web App

### Slide 2：這門課要學什麼

- 使用 LLM API，而不是只使用聊天網頁。
- 使用 Python 建立 AI 功能。
- 使用 Streamlit 建立 Web App。
- 使用文件、圖片、資料表與外部工具。

### Slide 3：這門課不做什麼

- 不訓練大型語言模型。
- 不進行本地端模型微調。
- 不要求高階 GPU。
- 不把重點放在單純 prompt 技巧。
- 不平均教所有 LLM 平台的語法。

### Slide 4：和上學期課程的銜接

上學期「生成式AI理論與實務」已涵蓋：

- 生成式AI基本觀念。
- Prompt 與應用場景。
- Skill 架構。
- Claude Desktop 使用 Skill。

本學期延伸：

- 將 Skill 思維融入自己開發的 Web App。

- 將作品部署成可使用的應用系統。

### Slide 5：聊天工具與 API 應用的差異

| 項目 | ChatGPT / Claude.ai / Gemini 網頁版 | LLM API 應用 |
|---|---|---|
| 使用方式 | 人直接對話 | 程式呼叫模型 |
| 輸入資料 | 手動貼上 | 可由系統自動取得 |
| 輸出處理 | 人閱讀 | 程式解析、儲存、轉換 |
| 整合能力 | 受平台限制 | 可接資料庫、檔案、工具、Web App |
| 適合情境 | 個人助理 | 資訊系統與企業流程 |

### Slide 8：什麼是 RAG

RAG 是讓模型根據外部資料回答問題的架構。

```text
文件資料
  → 切分段落
  → 建立 embedding
  → 查詢相關段落
  → 將段落放入 prompt
  → 模型根據資料回答
```

課程應用：

- 校園課程問答。
- 企業文件知識庫。
- 法規或制度查詢。
- 期末專題文件問答系統。

### Slide 9：什麼是 Agent

Agent 是讓模型根據任務目標，決定下一步要做什麼，並可使用工具完成多步驟任務。

```text
目標
  → 思考下一步
  → 選擇工具
  → 執行工具
  → 觀察結果
  → 繼續或完成
```

本課程會先教 Function Calling，再教簡易 Agent loop。

### Slide 10：Skill 在本課程中的角色

本課程不重教 Claude Desktop Skill 操作。

本課程重點是：

- 把 Skill 視為可重複使用的任務能力。
- 將任務說明、輸入格式、輸出格式、工具規則封裝成 Python 模組。
- 讓 Streamlit App 或 Agent 呼叫這些模組。

範例：

- 會議紀錄整理模組。
- 客服回覆草稿模組。
- 履歷分析模組。
- CSV 資料分析模組。

### Slide 11：什麼是 MCP

MCP 是讓 AI App 或 Agent 以標準方式連接外部工具與資料來源的協定。

本課程定位：

- 只做概念與示範。
- 不要求全班從零寫 MCP server。
- 可作為進階專題加分。

## 六、Git 與 GitHub 教學定位說明

### 本週定位：概念說明 + 帳號準備（15 分鐘）

Week 2–6 學生使用 Google Colab / Notebook，不需要版本控制。Week 7 切換至 VS Code + Streamlit 開發實際專案時，才是學習 Git 操作最自然的時機——學生有真實程式碼可以 commit，且可在同一堂課直接部署到 Streamlit Cloud，讓 Git 的用途立刻顯現。

因此本週 Git 章節的目標僅有兩項：

1. **讓學生理解為什麼本課程需要 Git 與 GitHub**
2. **完成帳號建立與工具安裝，Week 7 不再花時間處理環境**

完整 Git 實作教學（建立 repo、commit、push、部署）安排於 **Week 7**。

### 本週教師說明重點

向學生說明以下三件事即可：

- Git 是記錄程式修改歷史的工具，GitHub 是存放程式碼的雲端平台。
- 期中與期末專題都必須有 GitHub repository 與 README，部署也透過 GitHub。
- 本週只需要完成帳號建立，Week 7 會帶大家實際操作完整流程。

### 本週帳號準備清單

學生課後應完成：

1. 前往 [https://github.com](https://github.com) 建立帳號。
2. 前往 [https://desktop.github.com](https://desktop.github.com) 下載並安裝 GitHub Desktop。
3. 登入 GitHub Desktop，確認帳號連線正常。
4. 若使用學校電腦，確認是否能安裝軟體或登入個人帳號。

### 本週不要求

- 不要求建立 repository。
- 不要求操作 commit 或 push。
- 不要求理解 branch、merge、pull request。
- 不要求安裝 Git 命令列（GitHub Desktop 已內建）。

## 七、教師講稿重點

### 開場

這門課不是教大家如何訓練一個大型語言模型，而是教大家如何把大型語言模型接進資訊系統。未來在企業裡，多數資管背景的人不一定會自己訓練模型，但很可能需要把模型 API、資料庫、文件、Web 系統與企業流程整合起來。

### 課程定位

本課程的核心能力是 AI application development。你們會從 Python API 呼叫開始，逐步做到 Web App、RAG、圖片理解、工具呼叫與 Agent。期末要完成的是一個別人可以打開網址使用的應用，而不是只在 Notebook 裡跑一段程式。

### 平台選擇

課程主線使用 OpenAI API，原因是教材、範例、生態與初學者支援較完整。Claude 與 Gemini 不會被忽略，但會放在比較與自主學習階段，避免一開始被不同平台語法分散焦點。

### Skill 銜接

上學期已經學過 Skill 架構與 Claude Desktop 使用方式。本學期會把 Skill 的思維放進應用開發：如何把一個任務能力變成可重複使用的 Python 模組，讓 Web App 或 Agent 呼叫。

### 期末期待

期末專題不是比誰用最多模型，而是看能不能解決一個明確問題。好的專題應該有清楚的使用者、清楚的資料來源、穩定的操作介面、合理的錯誤處理，以及對成本與限制的說明。
