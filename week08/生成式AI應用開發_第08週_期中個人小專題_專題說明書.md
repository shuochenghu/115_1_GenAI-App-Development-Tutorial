# 生成式AI應用開發 第 08 週 期中個人小專題

> 本週主題：期中個人小專題  
> 實作平台：OpenAI API + Streamlit + GitHub  
> 課程定位：整合第 3 到第 7 週能力，完成一個可展示的小型 LLM Web App。

---

## 一、本週目標

第 8 週不是新 API 教學週，而是把前面幾週的能力整合成一個小型專題。

完成本週後，你應該能夠：

1. 用 Streamlit 建立一個可操作的 AI Web App。
2. 用 OpenAI Responses API 完成一個明確任務。
3. 設計 structured output，讓 AI 結果能被程式穩定讀取。
4. 用 `.env` 或 Streamlit Secrets 管理 API key。
5. 建立 GitHub repository，並用 README 說明專題目的、安裝方式與操作流程。
6. 用 3 到 5 分鐘展示你的 App 解決什麼問題、如何操作，以及 AI 回傳結果是否合理。

---

## 二、你要繳交什麼

期中個人小專題佔學期成績 20%。

最低繳交內容：

| 項目 | 要求 |
|---|---|
| Streamlit App | 必須可以用 `streamlit run app.py` 執行 |
| OpenAI API | 至少有一個主要功能會呼叫 OpenAI API |
| 結構化輸出 | 至少有一個功能使用「真 structured output」；建議用 JSON Schema + `strict`，也可用 Pydantic `responses.parse()` |
| API key 管理 | 不得把 API key 寫死在程式碼中，也不得上傳到 GitHub |
| GitHub repository | 專題程式需放在 GitHub repository |
| README | 說明專題目的、安裝方式、執行方式、功能流程與限制 |
| Demo | 課堂進行 3 到 5 分鐘展示 |

加分項目：

- 部署到 Streamlit Community Cloud。
- 支援 streaming 回覆。
- 有清楚的錯誤處理與輸入長度限制。
- 顯示使用限制、成本提醒或隱私提醒。
- UI 流程清楚，使用者不需要看程式碼也知道怎麼操作。

安全紅線：

- API key 不得寫死在 `app.py`、README、notebook 或任何會上傳 GitHub 的檔案中。
- 如果 API key 被推上 GitHub，屬重大安全問題，會列為重大扣分；不是只扣「安全與成本」項目的幾分。
- 本次結構化輸出要求不能用「請用 JSON 格式回答」這種自由文字提示充數。最低標準是 JSON Schema + `strict`，或 Pydantic `responses.parse()`。

## 三、建議題目

你可以從以下三個方向選一個，也可以自訂題目。自訂題目仍需符合最低要求。

| 題目 | 適合情境 | 建議輸入 | 建議輸出 |
|---|---|---|---|
| AI 摘要器 | 將長文、課堂筆記、會議記錄整理成重點 | 一段文字或文字檔 | 標題、摘要、重點、關鍵字、待辦事項 |
| 履歷助手 | 幫使用者整理履歷內容並提供修改建議 | 履歷文字、應徵職缺 | 技能摘要、優勢、缺口、修改建議 |
| 客服分類器 | 將客服訊息分類並產生處理建議 | 客戶訊息 | 類別、急迫程度、摘要、是否需人工處理、回覆草稿 |

## 五、Starter 專案使用方式

本週提供 `week08_midterm_starter/` 作為起點。你不需要從空白資料夾開始。

建議流程：

1. 複製 `week08_midterm_starter/`，改成自己的專題資料夾名稱。
2. 依照 `README.md` 建立 `.env`。
3. 執行：

```bash
pip install -r requirements.txt
streamlit run app.py
```

4. 打開 `app.py`，依序完成 TODO：

| TODO 區塊 | 你要完成什麼 |
|---|---|
| `PROJECT_TITLE` | 改成你的 App 名稱 |
| `PROJECT_DESCRIPTION` | 用一句話說明 App 解決什麼問題 |
| `SYSTEM_PROMPT` | 設計 AI 的角色、規則與限制 |
| `DEFAULT_SCHEMA` | 設計 structured output 欄位 |
| `build_prompt()` | 把使用者輸入整理成清楚的 prompt |
| `render_project_form()` | 設計 Streamlit 表單 |
| 結果顯示區 | 決定要用文字、JSON、表格或多個區塊顯示結果 |

## 十、常見問題

### Q1：一定要部署嗎？

不一定。最低要求是 GitHub repository 加上可本機執行的 Streamlit App。部署到 Streamlit Community Cloud 是加分。

### Q2：可以不用 Structured Outputs 嗎？

不可以。本次期中專題要求至少有一個真 structured output。建議主線使用 JSON Schema + `strict`，因為這和第 6 週正式教材銜接最直接；已熟悉 Pydantic 的同學也可以使用 `responses.parse()`。

只在 prompt 中寫「請用 JSON 回答」不算完成 structured output，因為程式端沒有明確 schema 契約，也較難穩定解析。

### Q3：可以做聊天機器人嗎？

可以，但必須有明確任務。例如「課程諮詢機器人」、「履歷修改助理」、「客服回覆助理」。如果只是一般聊天，不容易展現專題價值。

### Q4：可以用老師的範例摘要器直接繳交嗎？

不可以。教師 demo 是完成度參考，你的專題必須有自己的題目、prompt、輸入設計、輸出欄位與 README。

### Q5：可以把 API key 放在 README 或 app.py 嗎？

不可以。API key 只能放在本機 `.env` 或 Streamlit Secrets。上傳到 GitHub 的只能是 `.env.example` 或 `secrets.example.toml`。
