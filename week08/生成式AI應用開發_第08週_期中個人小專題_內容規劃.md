# 生成式AI應用開發 第 08 週 教材內容規劃

> 主題：**期中個人小專題（Midterm Individual Mini-Project）**
> 平台：OpenAI API + Streamlit + GitHub
> 本檔為教材製作前的內容規劃，已整合 Claude 草案與正式課程主線建議。

---

## 一、教學定位

| 項目 | 內容 |
|---|---|
| 主題 | 期中個人小專題（Midterm Individual Mini-Project） |
| 平台 | OpenAI API + Streamlit + GitHub |
| 性質 | **整合週 / 專題啟動週**，非新 API 教學 |
| 佔比 | 期中個人小專題 = 學期 20% |
| 環境轉捩點 | 大綱明訂「第 8 週之後以 VS Code + GitHub 專題形式進行」 |
| 前置技能 | 第 3–7 週：OpenAI API、Prompt Engineering、Streaming、Structured Outputs、Streamlit、Git / 部署 |

本週不再大量加入新 API，而是整合前 7 週技能，讓每位學生獨立完成一個可 Demo 的小型 LLM Web App，並正式從 Colab / Notebook 過渡到 VS Code + 本機 Python + GitHub 專案開發。

## 二、正式教材策略

第 8 週採「**專題說明書 + starter 專案骨架 + 教師 demo 範例**」為主，不優先製作學生版 / 教師版 notebook pair。

原因：

1. 課程大綱已將第 8 週之後定位為 VS Code + GitHub 專案開發。
2. 第 7 週已建立 `week07_streamlit_app/`，第 8 週應讓學生改造實際專案，而不是只在 notebook 裡練習。
3. 期中專題需要 GitHub repository、README 與可執行 App，Markdown 說明書比 notebook 更貼近繳交形式。
4. 若之後仍需要 notebook，可做成教師講解備忘或專題規劃 worksheet，但不作為主要教材產出。

### 第二優先：學生 starter 專案

2. **`week08_midterm_starter/`**
   - 對齊**課堂實際使用的 Codex 版** `week07_streamlit_app/` 的 helper 命名（`get_secret()` / `create_client()` / `ask_ai()` / `stream_ai()`）。
   - 不使用 `_claude` 命名，因為這會是正式課程 starter（需同步更新命名規則例外，見第十一節）。
   - `app.py` 保留 TODO，引導學生完成自己的專題。
   - **結構刻意比 Codex week07 更完整**：Codex `week07_streamlit_app/` 只有 `app.py` / `requirements.txt` / `README.md` 三檔；starter 另補 `.env.example` / `.gitignore` / `.streamlit/`，讓期中專題一開始就具備可上 GitHub 與可部署的完整骨架。

建議結構：

```text
week08_midterm_starter/
├── app.py
├── requirements.txt
├── .env.example
├── .gitignore
├── README.md
└── .streamlit/
    └── secrets.example.toml
```

`app.py` 建議保留的 TODO：

- `PROJECT_TITLE`：專題名稱
- `PROJECT_DESCRIPTION`：一句話說明 App 解決什麼問題
- `SYSTEM_PROMPT`：角色與回答規則
- `build_prompt()`：把使用者輸入整理成 LLM prompt
- `ask_ai()`：呼叫 OpenAI API（沿用 Codex week07）
- `extract_structured()`：結構化輸出骨架（**第 6 週概念，week07 專案沒有，需 starter 自行內建**）
- Streamlit form：建立輸入區與送出按鈕
- result rendering：顯示文字結果或 JSON 結構化結果
- safety / cost note：輸入長度限制、API 成本與隱私提醒

### 第三優先：教師 demo 範例

## 五、期中專題最低要求

期中專題是縮小版期末專題，要求應足夠明確，但不要超過一週可完成的範圍。

1. 一個可操作的 Streamlit Web App。
2. 至少一個 OpenAI API 功能。
3. 至少一項 **Structured Outputs** 或等價的結構化結果設計。
4. 使用 `.env` / `st.secrets` 管理 API key，不得寫死在程式碼中，不得上傳 GitHub。
5. GitHub repository + README。
6. 可進行 3–5 分鐘期中 Demo。

加分項目：

- 部署到 Streamlit Community Cloud。
- 支援 streaming 回覆。
- 顯示 API 成本估算或 usage。
- 有清楚錯誤處理與輸入長度限制。
- UI 有明確的使用流程與結果區塊。

## 六、三個參考題目方向

對應課程大綱：「AI 摘要器、履歷助手、客服分類器」。

| 題目 | 核心技能 | 結構化輸出範例 | 適合程度 |
|---|---|---|---|
| AI 摘要器 | 檔案上傳、摘要、條列重點、可選 streaming | `{"title": ..., "summary": ..., "key_points": [...], "keywords": [...]}` | 最適合入門 |
| 履歷助手 | 文字抽取、改寫、建議生成 | `ResumeInfo`：姓名、技能、經歷、優勢、修改建議 | 適合想做職涯工具者 |
| 客服分類器 | 分類、急迫程度、回覆草稿 | 類別、優先級、摘要、是否需回覆、建議回覆 | 最能連接商業應用 |

學生可自訂題目，但必須先填寫 App 設計表：

| 欄位 | 說明 |
|---|---|
| App 名稱 | 專題名稱 |
| 目標使用者 | 誰會用這個工具 |
| 使用情境 | 什麼時候需要它 |
| 使用者輸入 | 文字、檔案、表單欄位等 |
| AI 輸出 | 摘要、分類、建議、JSON 等 |
| 風險與限制 | 隱私、幻覺、成本、錯誤分類等 |

## 七、評分 Rubric 建議

| 項目 | 比重 | 評分重點 |
|---|---:|---|
| 功能完整度 | 30% | App 可執行，能完成一個明確任務，主要流程不會中斷 |
| LLM 使用品質 | 25% | prompt 清楚，輸出穩定，有基本限制；能使用結構化輸出或清楚的輸出格式 |
| Streamlit UI | 20% | 表單、結果顯示、錯誤提示清楚；使用者不用看程式碼也知道如何操作 |
| 安全與成本 | 10% | API key 未外洩；有 `.env` / secrets；有輸入長度、成本或隱私提醒 |
| README / Demo | 15% | README 說明完整；能在 3–5 分鐘內展示問題、操作流程與結果 |

## 八、模型與環境

- OpenAI 版本預設模型：`gpt-5.4-mini`，可用 `OPENAI_MODEL` 覆蓋。
- secrets 採雙軌：
  - 本機：`.env` + `python-dotenv`
  - 雲端：Streamlit `st.secrets`
- 統一使用 `get_secret()` 讀取 API key 與模型名稱。
- starter 專案沿用**課堂使用的 Codex 版** `week07_streamlit_app/` helper（已核對 `app.py`）：
  - `get_secret()`（`app.py:10`）
  - `create_client()`（`app.py:22`）
  - `ask_ai(user_input, system_prompt=...)`（`app.py:30`）
  - `stream_ai(user_input, system_prompt=...)`（`app.py:42`）
  - `extract_structured()`：**非 week07 helper，源自第 6 週結構化輸出，由 starter 自行內建**

## 九、與第 7 週銜接方式

第 8 週不從零開始，而是讓學生理解如何從第 7 週範例改造成自己的 App。

| 第 7 週能力 | 第 8 週使用方式 |
|---|---|
| `st.text_area()` / `st.form()` | 建立專題輸入表單 |
| `ask_ai()` | 完成主要 LLM 功能 |
| `stream_ai()` / `st.write_stream()` | 加分：串流顯示 |
| `st.session_state` | 保存使用紀錄或多次結果 |
| `.env` / `st.secrets` | 管理 API key |
| `.gitignore` | 避免上傳秘密檔 |
| `README.md` | 專題說明與繳交文件 |
